# Language Policy Plan

## Objective
Standardize the source code, logging terminology and user interface interactions strictly onto the English language layout.

## Requirements
- All technical documentation, execution phases, exceptions, logs and telegram text must be written in English language natively.
- Ensures global cross-readability across maintainers and logs tools.

## Steps
1. Identify all non-English prints or messages.
2. Translate texts explicitly.
3. Update `messages.py` accordingly.
